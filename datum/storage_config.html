storage_config
/datum/storage_config

Computes atom_storage values based on the initital contents of the storage. Use this only when the initial contents are extremly varied such that accurate values cannot be computed before hand e.g. if the items are randomly generated or you have huge list of stacks & stuff whose weights cannot be calculated correctly so setting the max_total_storage is not feasible and so on

Don't use this if you can manually compute the items weights/counts & set max_total_storage or other values correctly yourself as it can lead to balance implications in game e.g. if your storage initially carries a large item and you want to set max_specific_storage to that item weight then it will allow all others items less than or equal to that item weight which may not be ideal.

In conclusion use this when the initial contents are too complex for mortal comprehension so you make computing the storage values automatic

Vars

compute_max_item_countComputes number of items in the storage's initial contents & sets max_slots to that value
compute_max_item_weightComputes the item that has the highest weight in the storage's initial contents & sets max_specific_storage to that value
compute_max_total_weightComputes the total weight of the storage's initial contents & sets max_total_storage to that value
contents_are_exceptionsSets exception_hold to the storage's initial contents meaning they can be carried without max_specific_storage restrictions
whitelist_content_typesSets the storage's initial contents as the only items(i.e. sets can_hold) that can be carried by the storage

Procs

resetResets all values to their defaults
