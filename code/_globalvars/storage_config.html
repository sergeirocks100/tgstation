code/_globalvars/storage_config.dm

/datum/storage_configComputes atom_storage values based on the initital contents of the storage. Use this only when the initial contents are extremly varied such that accurate values cannot be computed before hand e.g. if the items are randomly generated or you have huge list of stacks & stuff whose weights cannot be calculated correctly so setting the max_total_storage is not feasible and so on
